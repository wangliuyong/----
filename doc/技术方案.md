结合你的技术栈 Next.js + NestJS，我们把原有 Module Federation 微前端方案全面替换为原生 Web Components。
Web Components 是浏览器原生标准（Custom Elements + Shadow DOM + HTML Template），无需第三方微前端框架，具备：
- 天然 DOM / 样式隔离，彻底解决样式污染
- 跨框架兼容（Next/React、Vue、原生JS 都能互相嵌入）
- 轻量无侵入、独立构建/部署/迭代，完全满足个人站微前端诉求
- 搭配 Lit（Google 官方 Web Components 框架）大幅简化开发（替代原生繁琐写法，推荐使用）
整体架构：
Next.js（主基座/外壳） + 多个独立 Web Components（业务子模块） + NestJS（统一后端接口）

3.1.2 编写自定义组件 & 业务逻辑
src/main.ts（组件入口，注册自定义元素）
import { LitElement, html, css } from 'lit'
import { customElement, property, state } from 'lit/decorators.js'

// 后端接口地址
const API_BASE = 'http://localhost:3001/api'

// 定义首页 Web Component
@customElement('wc-home')
export class WcHome extends LitElement {
  // 接收 Next 主应用透传的主题：light / dark
  @property({ type: String }) theme = 'light'

  // 组件内部状态：精选文章、精选项目
  @state() articles: any[] = []
  @state() projects: any[] = []

  // 样式：Shadow DOM 内部样式，天然隔离
  static styles = css`
    :host {
      display: block;
      width: 100%;
    }
    .container {
      padding: 2rem 0;
    }
    .intro {
      text-align: center;
      margin-bottom: 4rem;
    }
    .card-list {
      display: grid;
      grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
      gap: 1.5rem;
      margin-top: 1rem;
    }
    .card {
      border: 1px solid #e5e7eb;
      border-radius: 8px;
      padding: 1rem;
    }
    /* 暗黑模式样式（根据外部传入的 theme 切换） */
    :host([theme="dark"]) .card {
      border-color: #374151;
      background: #1f2937;
      color: #f3f4f6;
    }
  `

  // 组件挂载后请求接口
  connectedCallback() {
    super.connectedCallback()
    this.fetchData()
  }

  // 请求 Nest 后端数据
  async fetchData() {
    try {
      const [artRes, proRes] = await Promise.all([
        fetch(`${API_BASE}/article/list`),
        fetch(`${API_BASE}/project/list`)
      ])
      this.articles = await artRes.json()
      this.projects = await proRes.json()
    } catch (err) {
      console.error('接口请求失败', err)
    }
  }

  // 渲染模板
  render() {
    return html`
      <div class="container">
        <!-- 个人简介 -->
        <section class="intro">
          <h1>Hi，我是一名全栈开发者</h1>
          <p>技术栈：Next.js + NestJS + Web Components | 分享技术与项目</p>
        </section>

        <!-- 精选博客 -->
        <section>
          <h2>精选博客</h2>
          <div class="card-list">
            ${this.articles.slice(0, 3).map(
              item => html`
                <div class="card">
                  <h3>${item.title}</h3>
                  <p>${item.summary || '暂无摘要'}</p>
                </div>
              `
            )}
          </div>
        </section>

        <!-- 精选项目 -->
        <section style="margin-top: 3rem;">
          <h2>精选项目</h2>
          <div class="card-list">
            ${this.projects.slice(0, 3).map(
              item => html`
                <div class="card">
                  <h3>${item.name}</h3>
                  <p>${item.desc}</p>
                  <small>技术栈：${item.techStack}</small>
                </div>
              `
            )}
          </div>
        </section>
      </div>
    `
  }
}

// 注册自定义 HTML 标签（全局生效）
declare global {
  interface HTMLElementTagNameMap {
    'wc-home': WcHome
  }
}

4.2 通用工具：动态加载 Web Component 脚本
新建 app/utils/loadWc.ts，封装懒加载脚本逻辑（避免首屏加载所有组件，提升性能）：
// 动态加载 Web Component 脚本
export async function loadWebComponent(url: string) {
  // 避免重复加载
  const exist = document.querySelector(`script[src="${url}"]`)
  if (exist) return

  return new Promise<void>((resolve, reject) => {
    const script = document.createElement('script')
    script.type = 'module'
    script.src = url
    script.onload = () => resolve()
    script.onerror = (e) => reject(e)
    document.body.appendChild(script)
  })
}
4.3 路由页面：嵌入 Web Component（以首页 / 为例）
所有路由页面都是客户端组件，逻辑统一：
1. 获取当前全局主题
2. 动态加载对应 Web Component 脚本
3. 渲染自定义标签，并透传 theme 属性
示例 1：首页 app/page.tsx
'use client'
import { useEffect, useState } from 'react'
import { useTheme } from 'next-themes'
import { loadWebComponent } from './utils/loadWc'

// Web Component 静态资源地址（本地/线上 CDN）
const WC_HOME_URL = 'http://localhost:4001/dist/wc-home.es.js'

export default function HomePage() {
  const { theme } = useTheme()
  const [loaded, setLoaded] = useState(false)

  useEffect(() => {
    // 加载首页组件脚本
    loadWebComponent(WC_HOME_URL)
      .then(() => setLoaded(true))
      .catch(console.error)
  }, [])

  if (!loaded) return <div className="text-center py-20">加载中...</div>

  // 渲染原生自定义标签，透传暗黑主题
  return (
    <wc-home theme={theme}></wc-home>
  )
}

五、主应用 ↔ Web Component 双向通信（原生标准）
Web Components 无框架通信，使用 HTML 属性 + 原生 CustomEvent，完全浏览器标准。
5.1 方向1：主应用 → 子组件（传值：主题、路由、配置）
上面已实现：Next 通过 theme 属性把全局暗黑状态传给子组件。
扩展：还可以传递标题、用户信息、接口地址等：
<wc-blog theme={theme} api-base="http://localhost:3001/api"></wc-blog>
子组件内通过 @property 接收：
@property({ type: String }) apiBase = ''
5.2 方向2：子组件 → 主应用（事件回调：留言成功、跳转路由）
场景：留言表单提交成功（wc-contact 派发事件）
1. 子组件（wc-contact） 派发自定义事件
// 留言提交成功后
handleSubmit() {
  // 派发事件，携带数据
  this.dispatchEvent(
    new CustomEvent('message-success', {
      detail: { msg: '留言提交成功' },
      bubbles: true,
      composed: true // 穿透 Shadow DOM
    })
  )
}
2. Next 主页面（/contact） 监听事件
useEffect(() => {
  const el = document.querySelector('wc-contact')
  if (!el) return

  // 监听子组件事件
  const handleMsg = (e: any) => {
    alert(e.detail.msg)
  }

  el.addEventListener('message-success', handleMsg)
  return () => el.removeEventListener('message-success', handleMsg)
}, [loaded])
关键配置：bubbles: true、composed: true → 让事件穿透 Shadow DOM，主应用可以捕获。

// 原生 Web Component 极简示例
class WcHome extends HTMLElement {
  constructor() {
    super()
    // 开启 Shadow DOM
    this.attachShadow({ mode: 'open' })
  }

  connectedCallback() {
    this.shadowRoot!.innerHTML = `
      <style>
        div { color: #333; }
      </style>
      <div>首页内容</div>
    `
  }
}

// 注册标签

十、常见踩坑 & 解决方案
1. Next SSR 警告：未知 HTML 标签
解决：所有使用自定义标签的页面必须加 'use client'，仅客户端渲染识别 Web Components。
2. Shadow DOM 样式不生效 / 全局样式穿透
解决：Shadow DOM 默认隔离样式，组件内样式写在组件内部 <style>；如需全局主题变量，使用 CSS 变量 传递。
3. 脚本重复加载
解决：loadWebComponent 工具内做脚本去重判断。
4. 暗黑模式不同步
解决：监听 Next theme 变化，动态更新组件 theme 属性。
5. 事件无法被主应用捕获
解决：派发 CustomEvent 时必须开启 bubbles: true + composed: true。

2. 技术链路：Next.js(外壳路由) + Lit+Web Components(业务模块) + NestJS(后端)，架构轻量化、易维护、易扩展。
3. 扩展能力：后续可任意新增 Vue/React/原生页面，只需封装为 Web Component 即可嵌入，微前端能力永久保留。
